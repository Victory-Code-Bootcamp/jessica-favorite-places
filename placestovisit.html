HTML in 100 Seconds by Fireship

  • html was created at CERN
  • within html an opening and closing element is content and sometimes other html elements which are called children
  • the opening tag may have one or more attributes which customize the appearance or behavior of the element
  • head and body elements are children of the html element
  • the head element includes things that are not shown on the webpage, for example the title
  • the title is shown on the webpage's tab
  • meta element has no closing tag
  • img tag is self-closing
  • the alt tag is the description of an img allowing accessibility for screen readers and robots
  • use the a or anchor tag for a link
  • the div tag is used to group elements together

CSS in 100 Seconds by Fireship

  • css stands for cascade style sheet
  • css makes black and white html visually appealing and more interesting
  • there will never be a css4. only updates to the current language
  • a selector (element) is used that is matched to a html document
  • followed by braces to create a declaration block which include properties that change the appearance of the element
  • each is known as a css rule
  • multiple rules can be applied to the same element
  • rules have a hierarchy. rules that are more specific will override the rules of their parents
  • thus, the order of your css rules matter!
  • only some styles will inherit the styles of their parent
  • the browser uses cascade, specificity, and inheritance to control which styles are applied to an element
  • a selector can target multiple html elements of a certain type, multiple related elements with a custom class, or a single element with an id
  • think of each html element like a box
  • the content is in the middle followed by padding, border, and a margin
  • use grid and flex layouts to control how the boxes flow together
  • css can also detect user interaction, for example, hover
  • you can add logic to your code with variables, custom properties, and media queries which can apply styles conditionally based on the size or type of device
  • css can also handle complex animation with transitions and key frames

VS Code in 100 Seconds by Fireship

  • free and open source platform for editing code
  • provides an integrated development environment to help you edit, analyze, and manage your code files efficiently
  • world's most popular code editor
  • VS Code analyzes the symbols inside a file to perform syntax highlighting
  • it automatically detects problems in your code
  • For example, if a variable is nnot used, it becomes more opaque
  • a red squiggley line appears if your code will not compile. Hover over it for more detail.
  • creates an outline of a codes' structure to quickly navigate to the line of code that you're looking for
  • professor coders utilize VS Code's intellisense to autocomplete the code
  • when the correct suggesting comes up press tab or enter to complete
  • you may also right-click your code to find a variety of options to analyze and refactor your code
  • ctrl + shift + P to bring up the command palette
  • with the command palette, you can quickly access any Vs Code command or any file in your project
  • many extensions are available to provide specialized tooling
  • the remote button (bottom left) can connect your editor to a remote github repository, docker container, or cloud server
  • you can also run VSCode in the cloud to access it from the browser
